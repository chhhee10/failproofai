## لوحة التحكم

| المتغير | الوصف |
|----------|-------------|
| `PORT` | منفذ لوحة التحكم (الافتراضي: `8020`) |
| `CLAUDE_PROJECTS_PATH` | تجاوز موقع مجلدات مشاريع Claude Code |
| `FAILPROOFAI_DISABLE_PAGES=policies,projects` | صفحات لوحة التحكم المفصولة بفواصل لإخفاؤها |
| `FAILPROOFAI_ALLOWED_DEV_ORIGINS` | المضيفون/عناوين IP المسموح لها بالوصول إلى موارد التطوير. نفس `--allowed-origins`. |

## تسجيل السجلات

| المتغير | الوصف |
|----------|-------------|
| `FAILPROOFAI_LOG_LEVEL=info\|warn\|error` | مستوى سجل الخادم (الافتراضي: `warn`) |
| `FAILPROOFAI_HOOK_LOG_FILE` | مسار ملف السجل المخصص للخطاف، أو `true` للقيمة الافتراضية (`~/.failproofai/logs/hooks.log`) |

## بيانات الاستخدام

| المتغير | الوصف |
|----------|-------------|
| `FAILPROOFAI_TELEMETRY_DISABLED=1` | تعطيل بيانات الاستخدام المجهولة |

## المصادقة

| المتغير | الوصف |
|----------|-------------|
| `FAILPROOF_API_URL` | تجاوز عنوان URL الأساسي للخادم الذي تستخدمه `failproofai auth` وحوار مصادقة لوحة التحكم. القيمة الافتراضية `https://api.befailproof.ai`؛ اضبطها على `http://localhost:8080` (أو في أي مكان آخر) عند تشغيل خادم api محلي. |
| `FAILPROOFAI_AUTH_DIR` | تجاوز موقع تخزين `auth.json` (الافتراضي: `~/.failproofai`). مفيد بشكل أساسي للاختبارات المعزولة. |

## مطالبة التشغيل الأول

| المتغير | الوصف |
|----------|-------------|
| `FAILPROOFAI_NO_FIRST_RUN=1` | تخطي المطالبة التي تعرض تثبيت السياسات عند استدعاء `failproofai` الأول الفارغ |

## نموذج اللغة (لتقييم السياسة)

| المتغير | الوصف |
|----------|-------------|
| `FAILPROOFAI_LLM_BASE_URL` | نقطة نهاية واجهة برمجة التطبيقات للنموذج (الافتراضي: `https://api.openai.com/v1`) |
| `FAILPROOFAI_LLM_API_KEY` | مفتاح واجهة برمجة التطبيقات لسياسات مدعومة بنموذج اللغة |
| `FAILPROOFAI_LLM_MODEL` | اسم النموذج (الافتراضي: `gpt-4o-mini`) |
